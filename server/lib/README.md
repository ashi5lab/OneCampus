# Shared Server Library

Cross-cutting helpers used by multiple modules, not tied to any single feature.

- **`permissions.js`** — Phase 7 role/permission system: `onec_role_permissions` seed data (`DEFAULT_ROLE_PERMISSIONS`), `seedDefaultPermissions(client)`, and `can`/`cannot`/`hasPermission(req, permission)`. Paired with `server/middleware/permissionGuard.js`.
- **`audit.js`** — `logAudit(req, action, details)`: fire-and-forget insert into `onec_audit_logs` (spec Part 11). Never throws — a logging failure must not fail the request it's logging. Currently called from: `permissionGuard.js` (every `403`, action `permission.denied`), `evaluations/controller.js` `recordScore` (`evaluation.score_recorded`), `learners/controller.js` and `instructors/controller.js` `remove` (`*.deleted`), `certificates/controller.js` `issue` (`certificate.issued`). Add more call sites as new sensitive actions (role changes) get built.
- **`refreshTokens.js`** — `issueRefreshToken`/`rotateRefreshToken`/`revokeRefreshToken`, backing the httpOnly-cookie refresh flow in `auth/controller.js`. Opaque random tokens (not JWTs), only a SHA-256 hash stored in `onec_refresh_tokens`, rotated (not reused) on every `/auth/refresh` call.
- **`refreshTokenCleanup.js`** — `scheduleRefreshTokenCleanup(intervalMs)`, called once from `server.js` at startup. Rotation/revocation only ever marks `onec_refresh_tokens` rows, never deletes them, so the table grows unbounded otherwise. Runs once immediately then every `intervalMs` (server.js uses 24h), purging rows across every tenant schema that are expired, or revoked more than a day ago (short grace period for investigating a stolen/replayed token before it's gone for good).
- **`authCookies.js`** — `setAuthCookies`/`clearAuthCookies`: the refresh-token + CSRF-token cookie pair, kept in one place so login and refresh set them identically.
- **`credentials.js`** — `generateUniqueUsername`/`generatePassword`/`withTenantPrefix`/`placeholderEmail`: the login page no longer collects a tenant domain, so every username is `"<tenant prefix>_<local part>"` instead (e.g. `qs_adam2345`), and `server/middleware/tenantResolver.js` resolves the tenant for `POST /auth/login` straight from that prefix. Used by `learners`/`instructors`/`staff`'s `create()` (auto-generates username+password from first name + registry/staff id) and `bulkUploadProcessor.js` (same, per row); `guardians.create()` and `platform.approveTenant()` use just `withTenantPrefix` since those two still take a manually-typed local username.
- **`ownLearner.js`** — `getOwnLearnerId(req)`: row-level self-scoping for the `learner` role. Returns the caller's own `onec_learners.id` (or `null` if they're not a learner).
- **`ownGuardianLearners.js`** — `getOwnGuardianLearnerIds(req)`: the `guardian`-role equivalent, resolving the ids of every learner linked via `onec_learner_guardian_map` (see `server/modules/guardianLinks`). Returns `null` for non-guardians, or an array (possibly empty).
- **`rowScope.js`** — `getScopedLearnerIds(req)`: combines the two above into what controllers actually call. Returns `null` (no scoping — admin/staff/instructor) or an array of learner ids to filter by (`learner_id = ANY($n)`). Used by `attendance`, `certificates`, `evaluations` (scores), and `kindergartenActivity` list endpoints.
